    Introduction

    Welcome to the One Step Software API Documentation.

    Authentication

    To authorize, use this code:

    curl https://app.onestepsoftware.com/organizations/your-organization/residents.json
  -H "Authorization: Bearer authtokenxxxxxxxxxx"

    Make sure to replace authtokenxxxxxxxxxx with your API key.

    One Step Software uses API keys to allow access to the API.

    One Step Software expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: Bearer authtokenxxxxxxxxxx

    You must replace authtokenxxxxxxxxxx with your personal API key.

    Residents/Clients

    Get Clients

    curl "https://app.onestepsoftware.com/organizations/your-organization/residents.json"
  -H "Authorization: Bearer authtokenxxxxxxxxxx"

    The above command returns JSON structured like this:

    [
  {
    "id": 1,
    "first_name": "John",
    "middle_name": "Fender",
    "last_name": "Doe",
    "status": "Current"
  },
  {
    "id": 2,
    "first_name": "John",
    "middle_name": "Fender",
    "last_name": "Doe",
    "status": "Current"
  }
]

    This endpoint retrieves all residents (residents are also called clients).

    HTTP Request

    GET http://app.onestepsoftware.com/organizations/your-organization/residents.json

    Get a Specific Client

    curl "http://app.onestepsoftware.com/organizations/your-organization/residents/2.json"
  -H "Authorization: Bearer authtokenxxxxxxxxxx"

    The above command returns JSON structured like this:

    {
  "id": 2,
  "first_name": "John",
  "middle_name": "Fender",
  "last_name": "Doe",
  "status": "Current"
}

    This endpoint retrieves a specific client.

    HTTP Request

    GET http://app.onestepsoftware.com/organizations/you-organization/residents/<ID>.json

    URL Parameters

    Parameter Description
    ID The ID of the client to retrieve

    Create a Client

    curl -X POST \
  "https://app.onestepsoftware.com/organizations/sober-homes/residents.json" \
  -H 'Authorization: Bearer authtokenxxxxxxxxxx' \
  -F 'resident[first_name]=John' \
  -F 'resident[middle_name]=Fender' \
  -F 'resident[last_name]=Doe' \
  -F 'resident[status]=Current' \
  -F 'resident[occupancies_attributes][0][moved_in_at]=2019-03-01' \
  -F 'resident[occupancies_attributes][0][moved_out_at]=2020-03-31'

    The above command returns JSON structured like this:

    {
  "id": 2,
  "first_name": "John",
  "middle_name": "Fender",
  "last_name": "Doe",
  "status": "Current"
}

    This endpoint creates a client.

    HTTP Request

    POST http://app.onestepsoftware.com/organizations/you-organization/residents.json

    URL Parameters

    Parameter Description
    resident[first_name] The first name of the client
    resident[middle_name] The middle name of the client
    resident[last_name] The last name of the client
    resident[status] The status (also known as category) of the client (ex. Current, Lead, or Alumni)
    resident[occupancies_attributes][0][moved_in_at] The admit date of the client
    resident[occupancies_attributes][0][moved_out_at] The discharge date of the client

    Errors

    The One Step Software API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The resource requested is hidden for administrators only.
    404 Not Found -- The specified resource could not be found.
    405 Method Not Allowed -- You tried to access a resource with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The resource requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many resources! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
